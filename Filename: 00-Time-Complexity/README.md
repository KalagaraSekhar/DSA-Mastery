# 📖 Overview

Time Complexity is a measure of the amount of time an algorithm takes to execute as the input size increases.

Understanding time complexity helps us

- Compare different algorithms
- Write optimized code
- Improve interview problem-solving
- Choose the best possible approach

# 📊 Common Time Complexities

| Complexity | Name | Example |
|------------|------|----------|
| O(1) | Constant | Array Access |
| O(log n) | Logarithmic | Binary Search |
| O(√n) | Square Root | Prime Checking |
| O(n) | Linear | Linear Search |
| O(n log n) | Linearithmic | Merge Sort |
| O(n²) | Quadratic | Bubble Sort |
| O(n³) | Cubic | Matrix Multiplication |
| O(2ⁿ) | Exponential | Recursive Fibonacci |
| O(n!) | Factorial | Brute Force Permutations |

---

# 📌 Important Rules

- Ignore constants.
- Ignore lower-order terms.
- Focus on the dominant operation.
- Worst-case complexity is usually considered.
- Space complexity includes auxiliary memory.

## 💡 Key Takeaways

- Time Complexity estimates how runtime grows with input size.
- Big O describes the upper bound of runtime.
- Big Ω describes the lower bound.
- Big Θ describes the tight bound.
- Always analyze the dominant operation.
- Ignore constants and lower-order terms in asymptotic analysis.
- Auxiliary space is different from input space.
- Optimizing time often increases space usage (time-space trade-off).
